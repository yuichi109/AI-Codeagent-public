trigger: /save
---

## 使い方

```
/save ファイル名.yml
```

コードブロック内のテキストを `write_file` で即座に保存する。

## 厳守事項

- **内容を一切変更しない**（インデント・コメント・空行・日本語すべてそのまま）
- **解説・確認・read_file による中身確認を行わない**
- `write_file` を1回呼ぶだけで完了。それ以上のツール呼び出しは禁止
- 完了後は「保存しました」の1行のみ返す

## 手順

1. ユーザーのメッセージからコードブロック（` ``` ` で囲まれた部分）の内容を抽出する
2. `/save` の直後に指定されたファイル名を取得する（例: `/save check.yml` → `check.yml`）
3. `write_file(path="check.yml", content=<抽出した内容>)` を呼ぶ
4. 「✅ check.yml を保存しました」とだけ返す
