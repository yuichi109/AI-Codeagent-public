## 目的

チャットをリセットしても次のチャットで状況を完全に引き継げるよう、
会話の内容を構造化して保存する。

## 使い方

```
/memory
/memory TEST1
```

引数でプロジェクト名を指定する。省略した場合は会話の文脈から推定する。

## 手順

1. 会話履歴からプロジェクト名を特定する（引数 > 文脈推定 の優先順）
2. 保存先: `<プロジェクト名>/memory/`
3. 会話全体を分析して以下のファイルに振り分けて保存する

### MEMORY.md（インデックス・必須）

```
## メモリインデックス
最終更新: YYYY-MM-DD

### 現在地
（今どのフェーズか・直前に何をしたか・次に何をするか）

### ファイル一覧
- [progress.md](memory/progress.md) — 実装の進捗・完了済みタスク
- [decisions.md](memory/decisions.md) — 設計判断・選択した理由
- [issues.md](memory/issues.md) — 未解決の問題・注意事項
```

### memory/progress.md（実装進捗）

完了したタスク・実装済みの機能・ファイル構成を記録する。
内容がない場合は作成しない。

### memory/decisions.md（設計判断）

「なぜそうしたか」の判断記録。次のチャットで同じ議論を繰り返さないための記録。
内容がない場合は作成しない。

### memory/issues.md（未解決・注意事項）

未解決の問題・ハマりポイント・やってはいけないことを記録する。
内容がない場合は作成しない。

## 更新ルール

- 既存ファイルがある場合は**上書きせず追記・更新**する（過去の記録を消さない）
- 各ファイルは肥大化したら古い完了済み内容を削除して整理する
- MEMORY.md の「現在地」は毎回必ず更新する

## 厳守事項

- 保存内容の解説・読み上げをしない
- 余計な作業（git add 等）をしない
- プロジェクト名が特定できない場合のみ確認する
- 成功したら「✅ memory を保存しました」とだけ返す
