## なぜこの機能が必要か

自律エージェント（メインチャット）は便利だが、同じミスを繰り返すことがある。
その場合にユーザー自身がコードを開いて内容を確認しながら、AIと一緒に問題を追いたい。

また、コードをゼロから書くときも人間は迷う。

- 書き始めの構文・ボイラープレートが思い出せない
- 前に作ったコードを開いたがコメントが不十分で意味が分からない
- 「この行、何してるんだっけ？」という疑問をその場で解消したい
- 変数名・構造を相談しながら決めたい

こういった場面では「隣に座っているプログラマー仲間」のような存在が欲しい。
エディタを離れず、開いているコードを共有文脈として、AIと往復できる場所。

---

## メインチャットとの住み分け

| | メインチャット | インラインチャット |
|---|---|---|
| 主体 | AI が自律的に動く | 人間がハンドルを握る |
| 作業範囲 | ファイル横断・GitLab連携・複数ステップ | 今開いているファイルに集中 |
| ツール呼び出し | あり（read_file / run_command 等） | なし（コード変換・説明・実行のみ） |
| 会話履歴 | セッション全体で保持 | ペインを閉じるまで保持 |
| 向いている作業 | 「main.pyのXXXをリファクタして」 | 「この10行にエラーハンドリングを追加して」 |

## インラインチャットでできること

| 操作 | 具体例 |
|---|---|
| 質問に答える | 「このimportいる？」「PowerShellのシングルクォートとダブルクォートの違いは？」 |

## 確定している設計方針

### 会話モデル
- **対話式（往復前提）**。1ショットではない。
- 会話履歴（`messages[]`）はJSメモリ内で保持。ペインを閉じるとリセット。
- 毎ターン「現在のエディタのコード（または選択範囲）」を文脈として自動付与。

### 文脈の渡し方
- コードを選択している → 選択範囲を文脈として優先
- 選択なし → ファイル全体を文脈とする
- AIが提案を適用したあとは、適用後の状態が次ターンの文脈になる

### AIの返答形式
- テキストのみ（説明・回答）でも、コードブロック（提案）でも、どちらでも良い
- コードブロックがある場合のみ「✓ 適用」ボタンを表示
- 適用するとエディタが更新される（差分はデコレーションでハイライト）

### バックエンド
- `POST /editor/chat` エンドポイントを新設
- ツールなし・シンプルなChatCompletion呼び出し
- リクエスト: `{ messages, current_code, language, filename, model }`
- レスポンス: `{ reply }` （自由形式テキスト、コードブロック含む場合あり）

## ユースケース一覧（検討中に出たもの）

1. **デバッグ支援**  
   自律エージェントが繰り返すミスをユーザーが発見し、コードを見ながら一緒に原因を特定・修正する。

2. **Ansible等の設定ファイルをゼロから書く**  
   「ここにAzureへ接続してリソースグループ作るやつを書いて」→確認→「この変数名を変えて」のループ。実行不要なことも多い。

3. **PowerShellスクリプトを書いて動かす**  
   「システム時刻を取得してファイル名にしてコピーするやつ」を書いてもらい、実行・確認・修正を繰り返す。

4. **コードの意味を確認しながら作業**  
   過去に書いたコードを開いたがコメントが不十分。「この行何してるの？」と聞きながら理解を深め、必要なら修正する。

5. **書き始めのサポート**  
   ゼロから書くときの「どうやって始めるんだっけ」をその場で解決。ボイラープレートや構文を相談しながら進める。
